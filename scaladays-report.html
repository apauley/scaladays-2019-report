A Few ScalaDays Talks

A Tour of Scala 3 (Keynote)

Martin Odersky (@odersky)

Summary: Some cool new features that makes Scala development easier and simpler for beginners, as well as many features that will help you when writing functional programs.

Techniques for Teaching Scala

Noel Welsh (@noelwelsh)

Summary: We should focus on developing our teaching skills. Here are some useful ways to do this in Scala.

Akka Streams to the Extreme

Heiko Seeberger (@hseeberger)

Summary: Akka Actors have many anti-patterns. Here is an example where doing distributed computation with Akka Streams can be better.

Massively Parallel Distributed Scala Compilation… And You!

Stu Hood (@stuhood)

Summary: Twitter is doing some amazing things to speed up compilation on a code base with millions of lines of Scala code.

Making Our Future Better

Viktor Klang (@viktorklang)

Summary: In future Future's will still immediately start running when created. Regardless, there are a few things that can be done to make an eager Future faster and better. Viktor said they did those things. And notably, they haven't mentioned cats-effect IO, Monix or ZIO.

Concurrent programming in 2019: Akka, Monix or ZIO?

Adam Warski (@adamwarski)

Summary: An impressive live-coding session where the same problem was solved, first with Akka actors and then with ZIO.

Pure Functional Database Programming‚ without JDBC

Rob Norris (@tpolecat)

Summary: Rob is the author of the most popular DB library for Scala, Doobie (using JDBC). JDBC is very old and blocking. Rob mentioned his new Skunk library, and how he used some of the existing pure-functional libraries (e.g. scodec and FS2) to make a direct TCP connection to PostgreSQL, and do some non-blocking DB queries.

ScalaClean - full program static analysis at scale

Rory Graves (@a_dev_musing)

Summary: If your analysis includes all of the code to be deployed, then ScalaFix can safely modify the code in interesting ways. For example, it can change method/class/package visibility to the minimum, or remove unused parameters.

Compiling to preserve our privacy

Manohar Jonnalagedda and Jakob Odersky.

Summary: Suppose you want to get answers from multiple data sources. But the data in each source can't be shared.

Real world example: certain organisations maintain spy satellites in orbit around the earth. None of them desire sattelite collisions. And none of them want to reveal the locations of their sattelites.

Interactive Computing with Jupyter and Almond

Sören Brunk (@soebrunk)

Summary:

Almond allows you to write Scala code in your Jupyter notebooks. The presentation was done entirely within Almond/Jupyter and featured live code editing from within the presentation.

All the fancy things flexible dependency management can do

Alexandre Archambault (@alxarchambault)

Summary: Coursier can do much more than just parallel downloads. Among other things it can install applications and help bootstrap a clean system with a Scala dev environment.
